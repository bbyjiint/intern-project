═══════════════════════════════════════════════════════════════════════════════
                    MAJOR UPDATES FROM PREVIOUS SCHEMA
═══════════════════════════════════════════════════════════════════════════════

✅ NEW: UniversityFaculty join table (Many-to-Many)
   - Links University and Faculty
   - Unique constraint on (universityId, facultyId)
   - CandidateUniversity references this pair

✅ UPDATED: Faculty model
   - Removed universityId field
   - Now global entity (unique by name, thname, code)
   - Can belong to multiple universities

✅ UPDATED: CandidateUniversity model
   - Changed facultyId → universityFacultyId
   - Now references UniversityFaculty pair
   - Supports multiple universities per candidate

✅ REMOVED: Direct University-Faculty relationship
   - Old: University (1) ──→ (N) Faculty
   - New: University (N) ──→ (N) Faculty [via UniversityFaculty]
